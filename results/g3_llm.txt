G3 — hidden-vs-verbal state injection INSIDE the LLM (Milestone G)
==================================================================
Script: scripts/g3_hidden.py (data /tmp/gps_slime/chess_twithstate_*.jsonl).
Model: Qwen3-1.7B, LoRA (r=16), 2 epochs, 3 seeds. Metric: held-out mean
completion NLL (lower = better). Three conditions, ONE custom loop (fair):
  none    = position only
  verbal  = position + the state as an English note (HumanLM-style)
  hidden  = position + the SAME state (5 parsed features) as a trained
            soft-prompt prefix (HiddenPrefixProjector), text identical to none.
Two fixes were required for a valid comparison (both are real methodology, not
tuning): use the CHAT TEMPLATE (Qwen3 is instruction-tuned; raw text made the
known verbal effect vanish), and NEAR-ZERO init the projector (a random prefix
before the chat tokens corrupts the sequence).

FINDINGS (honest)
-----------------
1. VERBAL state helps the LLM predict think-time -- robust, all 3 seeds
   negative (verbal-none mean -0.0050). This reproduces the known board-native
   "state -> timing" benefit in an actual LLM (the positive G2 result). (~half
   the published TRL magnitude -0.011; my per-example loop scores the number
   tokens differently -- the *comparison* is what's valid, not the absolute.)

2. HIDDEN does NOT beat verbal. hidden-verbal is +0.0034 on average (2/3 seeds
   favor verbal, 1 ~tied), and hidden's own benefit over no-state is marginal
   and inconsistent (hidden-none mean -0.0016, one seed ~0). So the LLM-native
   "hidden >> verbal" headline is NOT supported here.

3. WHY -- and the real result: the hidden-vs-verbal ordering is
   BACKBONE-DEPENDENT. The LLM understands the verbal note *semantically*
   (pretraining knows "tilt", "time pressure"), so text is the efficient
   channel; a from-scratch soft-prompt of the same numbers can't compete at
   this data/adaptation scale. Board-native (NO language prior) is the reverse:
   the richer hidden vector beats the lossy verbal dims (RQ6, -0.069/-0.117).
   So which channel wins depends on whether the model has a language prior.

IMPLICATION FOR THE FRAMING
---------------------------
The crisp "hidden >> verbal" evidence stays BOARD-NATIVE (RQ6). In the LLM the
honest contribution is: (a) injected state is usable by a real LLM (verbal
helps think-time), and (b) the channel ordering flips with the backbone's
language prior. Novelty #3 must be stated this way, not as "hidden >> verbal in
an LLM".

v2 (FOLLOW-UP, not yet run): give hidden the FULL trained *evolving* latent
(joint injector training) rather than the same 5 features as verbal. Only that
could let hidden carry MORE than words express and possibly overcome verbal's
language-prior advantage -- but effects here are tiny (~0.003-0.008 nats) and a
tie/negative is the likely outcome. Deferred pending a decision (low payoff).
